RTC Blacklist – User Privacy Policy

Effective Date: 1st March 2026

1. Introduction

This Privacy Policy explains how RTC Blacklist (“the Bot”) collects, uses, stores, and protects user information when interacting with the Bot on Discord.

This policy applies to individual users who interact with the Bot through commands, automated moderation systems, or enforcement systems.

2. Information We Collect

2.1 Discord User Information

  • Discord User ID
  • Discord User Avatar
  • Username and discriminator
  • Display name
  • Server ID(s) and names where actions occur

2.2 Moderation & Enforcement Data

  • Blacklist status
  • Blacklist reason (if provided)
  • ID of the moderator who executed the action
  • Timestamp of the action
  • Errors associated with the blacklist

2.3 Technical Data

  • Command usage logs
  • Error logs
  • Server IDs related to enforcement
  • Basic runtime/system diagnostics (if system commands are used)

The Bot does not collect:

  • Email addresses
  • Passwords
  • Direct message contents (unless sent directly to the Bot)
  • IP addresses
  • Payment information

3. How We Use Information

  • Global blacklist enforcement
  • Moderation logging
  • Security and anti-tamper monitoring
  • Preventing abuse of RTC-affiliated servers
  • Bot diagnostics and stability

Data is not used for advertising, profiling, or resale.

4. Data Storage

Blacklist data is stored in a secure JSON file or database.

Logs may be stored in designated logging channels restricted to admins.

Data is retained as long as necessary for moderation and enforcement purposes.

Access to stored data is restricted to the Bot owner.

5. Data Sharing

User data is not sold or shared with third parties.

However, blacklist enforcement may result in actions (such as bans) across multiple RTC-affiliated servers as part of the Bot’s intended functionality.

6. User Rights

  • Confirmation of blacklist status
  • The reason for blacklist placement
  • Review of stored moderation data
  • Removal from blacklist (subject to approval)

Requests must be made directly to the Bot owner.

7. Security

  • Owner-only administrative commands
  • Anti-tamper monitoring
  • Controlled logging channels
  • Restricted command access

While reasonable safeguards are in place, no system can guarantee absolute security.

8. Automated Enforcement

  • Ban blacklisted users upon joining affiliated servers
  • Log suspicious system or permission changes
  • Enforce restrictions without manual intervention

These actions are part of the Bot’s designed functionality.

9. Policy Updates

This Privacy Policy may be updated at any time. Continued use of the Bot constitutes acceptance of the updated policy.
